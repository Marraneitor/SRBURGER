Ideas priorizadas para implementar

Esta página es un backlog de mejoras (UX, diseño, operación, fidelización y seguridad). Cada tarjeta tiene un objetivo claro y un "cómo" corto (sin entrar en código).

1) Checkout más rápido (menos pasos)

UX

Reducir fricción: menos clics para terminar compra.

  • Autollenar teléfono/nombre desde perfil (si existe).
  • Mostrar resumen fijo del total y envío.
  • Validar campos en vivo y mostrar errores claros.

2) Estado del pedido tipo "línea de tiempo"

Tracking

En [tuenvio.html](tuenvio.html) mostrar pasos visuales (recibido → preparando → listo → en camino → entregado).

  • Mostrar hora estimada y última actualización.
  • Explicar qué significa cada estado.
  • Botón de WhatsApp "Necesito ayuda" con el número de orden.

3) Impresión de tickets (cocina)

Operación

Imprimir un ticket simple por pedido (para cocina y entrega).

  • Vista "ticket" (CSS print) con items, notas, total y dirección.
  • Botón "Imprimir" en Control de Envíos.
  • Formato grande (modo cocina).

4) Historial operativo + búsqueda rápida

Operación

Encontrar pedidos por teléfono, nombre o #orden en segundos.

  • Campo de búsqueda arriba de la lista.
  • Filtro rápido de hoy/ayer/semana.
  • Exportación simple a CSV.

5) Recompensas más claras en el checkout

Puntos

Que el cliente entienda cuándo gana y cuánto le falta para la siguiente recompensa.

  • Barra de progreso de puntos en el carrito.
  • Mensaje: "Con esta compra ganas X puntos" (solo si se paga).
  • Mostrar recompensa activa y cómo se aplica.

6) Cupones por código

Ventas

Crear códigos como SR10, ENVIOGRATIS, etc. con reglas simples.

  • Validez por fecha y usos máximos.
  • Porcentaje o monto fijo.
  • Restricción a monto mínimo.

7) Panel protegido (login admin)

Seguridad

Evitar que cualquiera en la red marque pedidos o acceda a datos.

  • Login con Firebase Auth para roles de staff.
  • Reglas Firestore por roles (cocina / repartidor / admin).
  • Eliminar el prompt de clave y usar sesión.

8) Notificaciones automáticas (sin abrir WhatsApp)

Infra

Enviar WhatsApp/SMS automático al cliente (confirmado / en camino / entregado).

  • Backend/Cloud Functions que observe cambios del pedido.
  • Plantillas de mensajes consistentes.
  • Control de reintentos y deduplicación.

9) Reportes (ventas / envío / tiempos)

Datos

Ver qué se vende, cuánto tardan pedidos y cuánto cuesta el envío promedio.

  • Top productos por semana.
  • Tiempo promedio: pending→ready→delivered.
  • Mapa de colonias más frecuentes (sin mostrar ubicación exacta).

10) Catálogo editable desde admin

Admin

Modificar menú/precios/extras sin tocar código.

  • Productos y extras en Firestore.
  • Historial de cambios (auditoría).
  • Botón publicar/ocultar productos.

Ideas de Diseño Web (para que se sienta más premium)

Son cambios de UI/UX visuales que mejoran la experiencia sin meter complejidad técnica innecesaria.

UI / UX

11) Jerarquía visual clara en cards (menos “ruido”)

UI

Que el ojo entienda rápido: nombre → precio → acción (botón).

  • Reducir texto secundario (descripción) y usar “ver más” solo si hace falta.
  • Precio más grande y consistente (mismo tamaño en todo el menú).
  • Botón con una sola intención clara (Personalizar vs Agregar).

12) Consistencia de componentes (se siente “app”)

UI

Un mismo patrón para botones, chips, badges, modales y inputs.

  • Definir 3 tamaños de botón: primary / secondary / ghost.
  • Un solo estilo de “badge” (Popular, Oferta, Ahorro, Nuevo).
  • Inputs con mismo alto, borde, y estados (focus/error/disabled).

13) Accesibilidad rápida (más fácil para todos)

A11y

Mejor legibilidad y menos errores (especialmente en móvil).

  • Tamaño mínimo de toque: botones de 44px y buen espaciado.
  • Contraste correcto en dark/light (texto y botones).
  • Mensajes de error cortos y específicos (“Falta tu colonia”).

14) Estados de carga / vacío (evita confusión)

UX

Cuando algo tarda, el usuario debe ver “qué pasa” y “qué sigue”.

  • Skeletons en imágenes y listas al abrir el menú.
  • Carrito vacío con CTA: “Explorar combos” / “Ver hamburguesas”.
  • Botones con estado “cargando…” al enviar pedido.

15) Microcopy que vende (texto corto que guía)

Copy

Textos pequeños en el lugar correcto aumentan conversiones.

  • En el checkout: “Tu pedido tarda 25–35 min aprox.”
  • En combos: “Elige tu hamburguesa” / “Puedes personalizar después”.
  • En envío: “Se calcula a $8/km, máximo 12km” (simple y visible).

16) “Sticky cart” discreto en móvil

Mobile

Que el cliente siempre tenga a mano el total y el botón de pagar.

  • Barra inferior con Total + Ver carrito (solo cuando hay items).
  • Evitar tapar botones (respeto a safe-area en iPhone).
  • Mostrar “+1 agregado” como feedback rápido.

17) Fotos “premium”: encuadre y consistencia

UI

No es solo “tener foto”, es que todas se vean coherentes.

  • Misma proporción (1:1) y fondo uniforme para hamburguesas.
  • Evitar fotos demasiado oscuras o con mucho ruido de fondo.
  • Una regla por producto especial (ej. Combo Dúo sin recorte).

18) Animaciones útiles (no por adorno)

UX

Micro-interacciones para confirmar acciones y reducir errores.

  • Al agregar: mini confirmación + vibración suave (mobile).
  • Al abrir modal: transición corta y consistente (220–300ms).
  • Respetar “prefers-reduced-motion” si está activo.

Checklist rápido (si quieres que se vea “pro”)

  • Spacing consistente (8/12/16/24) en todo.
  • Un solo estilo de iconos (misma familia y grosor).
  • CTA principal siempre visible (menú / carrito / pagar).
  • Estados: loading / empty / error bien definidos.
  • Textos cortos y orientados a acción (“Continuar”).
  • Contraste OK en dark mode (sin texto gris oscuro).
